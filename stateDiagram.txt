D) MT5 Executor Hardening (mt5_executor.py)
================================================================================

Fail-closed guards (สำคัญ)
- request_id required + dedup persistent state
- ai_confirm.approved ต้อง true (ไม่งั้น SKIP)
- decision ต้อง BUY/SELL เท่านั้น
- plan ต้องมี entry/sl/tp และเป็น number/side ถูกต้อง
- spread/cooldown/margin checks
- logging ทุกเหตุการณ์ลง logs/execution_orders.jsonl

Adaptive pyramiding policy (as of 2026-03-10)
- เป้าหมาย: ไม่ fix จำนวนไม้ แต่ “เพิ่มไม้เฉพาะตอนเทรนด์ไปทางถูก” (pyramiding)
- เงื่อนไขหลัก:
  - Regime ต้องอยู่ในชุดที่อนุญาต (default: TREND,EXPANSION)
  - Alignment_score >= threshold (default: 2)
  - ST direction ต้องไปทางเดียวกับ order (BUY:+1 / SELL:-1)
  - กันไล่ราคา: supertrend_distance_atr <= max (default: 3.2) — ใช้ทั้ง “ไม้แรก” และ “เพิ่มไม้”
  - เพิ่มไม้เมื่อราคาไปทางถูกจากไม้ล่าสุด >= step_atr * ATR (default: 0.8*ATR)
  - จำกัดด้วย margin_free แบบ dynamic + buffer และมี abs max positions กัน runaway
- Env knobs:
  - EXECUTION_PYRAMID_ENABLE (default 1)
  - EXECUTION_PYRAMID_STEP_ATR (default 0.8)
  - EXECUTION_PYRAMID_MIN_ALIGN (default 2)
  - EXECUTION_PYRAMID_ALLOW_REGIMES (default TREND,EXPANSION)
  - EXECUTION_PYRAMID_MAX_ST_DISTANCE_ATR (default 3.2)
  - EXECUTION_PYRAMID_MARGIN_BUFFER (default 1.6)
  - EXECUTION_ABS_MAX_POSITIONS (default 50)

Adaptive reverse policy (as of 2026-03-10)
- เป้าหมาย: หยุดการค้างสองฝั่ง (BUY+SELL) และ reverse แบบมีเงื่อนไข
- Flow:
  - ถ้าเจอ opposite positions:
    - ถ้า reverse disabled => block ด้วย opposite_buy_open/opposite_sell_open
    - ถ้า reverse enabled และ signal confirmed => close opposite ก่อน
    - หลังปิดแล้ว ยังต้องผ่าน anti-chase/position guards ก่อนส่ง order ใหม่
- Reverse confirm gates:
  - alignment >= 2
  - supertrend direction ไปทางเดียวกับ signal
  - regime in EXECUTION_PYRAMID_ALLOW_REGIMES
  - decision_votes (ถ้ามี) ต้องผ่าน EXECUTION_REVERSE_MIN_VOTES และไม่มี vote สวน
  - supertrend_distance_atr <= EXECUTION_REVERSE_MAX_ST_DISTANCE_ATR
- Env knobs:
  - EXECUTION_ADAPTIVE_REVERSE (default 1)
  - EXECUTION_REVERSE_MIN_VOTES (runtime tuned to 3)
  - EXECUTION_REVERSE_MAX_ST_DISTANCE_ATR (runtime tuned to 2.8)
